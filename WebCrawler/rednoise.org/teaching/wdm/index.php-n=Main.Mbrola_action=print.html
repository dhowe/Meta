Mbrola on Windows 

  - Download the PC/DOS package and 1 mbrola voice (eg, 'us1') from here:

    http://tcts.fpms.ac.be/synthesis/mbrola/mbrcopybin.html

  - Create a home directory for mbrola: eg, c:\MbrolaHome
  - Unpack the mbrola zip into the new directory
  - Make a copy of mbrola.exe and name it 'mbrola'
  - Open a command-prompt (start->run->cmd) 

  - Navigate to your mbrola directory which should look like this:

    C:\Documents and Settings>cd c:\MbrolaHome

    C:\MbrolaHome>dir

      Directory of C:\MbrolaHome
      02/25/2009  11:38 PM    <DIR>          mbr302a
      02/25/2009  02:45 PM            90,112 mbrola
      02/25/2009  02:45 PM            90,112 mbrola.exe
      02/25/2009  04:56 PM            29,623 readme.txt

  - Unzip your voice directory into the mbrola home directory:

    C:\MbrolaHome>dir

      Directory of C:\MbrolaHome
      02/25/2009  11:38 PM    <DIR>          mbr302a
      02/25/2009  02:45 PM            90,112 mbrola
      02/25/2009  02:45 PM            90,112 mbrola.exe
      02/25/2009  04:56 PM            29,623 readme.txt
      02/25/2009  11:55 PM    <DIR>          us1


  - cd into your voice directory

    C:\MbrolaHome>cd us1

  - Then run this test command:

    C:\MbrolaHome\us1>..\mbrola us1 TEST/alice.pho test.wav

  - If it succeeds you should see test.wav file in the current directory:

    C:\MbrolaHome\us1>dir

      Directory of C:\MbrolaHome\us1
      02/25/2009  06:37 PM             1,648 license.txt
      02/25/2009  12:03 PM    <DIR>          TEST
      02/25/2009  11:59 PM           485,804 test.wav
      02/25/2009  05:51 PM         7,238,094 us1
      02/25/2009  03:02 PM             4,125 us1.txt
      02/25/2009  12:02 PM               503 us1mrpa

  - Listen to it with an audio-player (e.g., Windows Media Player).
    If it sounds ok, you're ready to test with RiTa.

  - Try the sketch below from within Processing:

  import rita.*;

  void setup()
  {
    size(200,200);
    RiSpeech rs = new RiSpeech(this);
    rs.setMbrolaBase("c:\\MbrolaHome\\");
    rs.setVoice("mbrola_us1");
    rs.speak("hello");
  }

  void draw() {
    background(255);
  }

Mbrola on Mac OsX

[NOTE: FreeTTS + Mbrola seems to be broken on Mac OSX -- if you are interested in using it for a project, let me know and we can do some further investigation.] 

  - Download the MacOSX package and 1 mbrola voice (eg, 'us1') from here:

    http://tcts.fpms.ac.be/synthesis/mbrola/mbrcopybin.html

  - Create a home directory for mbrola: eg, /Users/jane/MbrolaHome/
  - Unpack the mbrola zip into the new directory
  - Make a copy (or symlink) of mbrola-darwin-ppc and name it 'mbrola'
  - Open a terminal (/Applications/Utilities/Terminal) 

  - Navigate to your mbrola directory which should look like this:

  ...

  - Unzip your voice directory into the mbrola home directory:

  ...

  - cd into your voice directory

    /Users/jane/MbrolaHome/$ cd us1

  - Then run this test command:

    /Users/jane/MbrolaHome/us1$ ../mbrola us1 TEST/alice.pho test.wav

  - If it succeeds you should see test.wav file in the current directory:

    /Users/jane/MbrolaHome/us1$ ls

    ...  

  - Listen to it with an audio-player (e.g., iTunes).
    If it sounds ok, you're ready to test with RiTa.

  - Try the sketch below from within Processing:

  import rita.*;

  void setup()
  {
    size(200,200);
    RiSpeech rs = new RiSpeech(this);
    rs.setMbrolaBase("/Users/jane/MbrolaHome/");
    rs.setVoice("mbrola_us1");
    rs.speak("hello");
  }

  void draw() {
    background(255);
  }
