ASSIGNMENTS

Assignment #1 (Due Mon. Feb 4)

1. TOOLS AND SETUP
  1. Download & install the Processing environment: http://processing.org
  2. Verify you can run the examples in the Sketchbook:
    Menu: File->Sketchbook->Examples
  3. Visit the exhibition section on the processing site: http://processing.org/exhibition
  4. Visit and bookmark the (extended) Processing reference: http://processing.org/reference/index_ext.html
  5. Visit and bookmark the Java API reference here: http://java.sun.com/j2se/1.4.2/docs/api/
  6. Read the short chapter by Greenberg on 'Creative Coding'
2. DEVELOPING A CRITICAL VOCABULARY
Select 2 Processing projects (the Exhibition section is a good place to start) that you like and be prepared to talk about how they work and what you like about them. Be sure to address specific conceptual, technical, and aesthetic aspects of the work. (All selected projects should have a link to their source code)
3. PROGRAMMING REVIEW/CONCEPTS
Read the ProgrammingConcepts pages on the topics below and make sure you understand the concepts (Much of this may be review). Come to class with at least 2-3 questions that came up for you during the reading.
  1. Operators, Variables, Arrays
  2. Conditionals, Program Flow, Loops,
  3. Functions, Classes and Objects


UPCOMING ASSIGNMENTS       (to start thinking about)


'3 MINI-PROJECTS'    (chosen from below)

Recombination Based on our discussion of recombination and the examples we've looked at, create a recombinant literary work. While you are free to choose any form, one of the 3 poetic forms we've discussed (haiku, sonnet, sestina) is probably a good place to start.

Feature Analysis Prepare a sketch to present that uses feature extraction to generate new literary content. Features that may be extracted via the RiTa library include: word, pos, stress, syllables, and phonemes (technical details to be discussed in lab). Find a way to use this data to make something creative/personal/expressive.

Grammar-based generation Prepare a sketch to present that uses your own grammar. (technical details to be discussed in lab). Find a way to use this technique to make something creative/personal/expressive. First iterations should use only the basic context-free grammar. Discuss any limitations you encounter in your presentation.

Text-mining Prepare a sketch to present that uses the web-scraping and text-mining techniques we've discussed to generate new literary content. Try to take advantage of the rapidly changing nature of web data sources to create a work that evolves over time. If you like, use the web-parsing, rss, and sentence extraction components in the RiTa library. Find a way to use this data to make something creative/personal/expressive (Note: this sketch can be submitted as an application (test, zip, and email).

Ontology-based generation (Wordnet) Prepare a sketch to present that make creative use of an ontology (a hierarchically structured lexicon). Wordnet is a good choice here (see the Rita.Wordnet library) but there are other options: Verbnet, Framenet, Propbank, OpenCyc, or even . Find a way to use one or more of these resources to make something creative/personal/expressive. Include in your presentation any limitations in the data sources you selected.



FINAL PROJECTS

Final projects will be discussed in 1-on-1 meetings and should build on core concepts in the course. These will be presented formally in the last week of classes in an 'artist-talk' format complete with documentation on all stages of the process (initial ideas, sketches/images, storyboards, recordings, prototypes, writing, etc.) you will be required to hand this in as part of the final project.)

Remember to consider the use of color, space, image, animation and sound (samples, text-to-speech, or you reading) as means to extend the range of the piece in an organic/cohesive fashion. Be prepared to present your ideas and or work-in-progress next week.

Although project are due at end-of-semester, it is highly encouraged that you start right away and take full advantage of of all course resources...

FINAL RESPONSE PAPER

Write a short (3-5 page) response to your experience (in this class) of programming for digital art and literature. Focus here on ways this might (or might not) affect your future practice. Feel free to discuss whatever topics were of personal interest and what, if any, future digital art projects you might imagine. This should be delivered with the complete documentation of your project.
