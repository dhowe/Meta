Regular Expressions Exercise (3 parts)

1) Write a regular expression that, given an arbitrary HTML page, extracts all of the valid 'sentences' and returns them in an a List (you can use an ArrayList). A 'sentence' (for the purpose of this exercise) will be defined as a sequence of at least 3 words, beginning with a capital letter & ending with a period, question mark, or exclamation point. Feel free to use the sample code from the reading as a template. You should test (use the fetch() methods from the RiHtmlParser if you like) with various web pages (don't use only one!), verifying that:

  1. all extracted items are real sentences,
  2. no real sentences are missed, and
  3. no extracted sentences contain bad (non-textual) characters (<, >, ~, ^, *, [, ], {, }, | etc.)

Adapt your expression to handle 'fringe' cases, e.g., abbreviations, punctuation inside quotations, entities (&copy;) and html markup (<b>), etc. 

    boolean stripTags = false; // true to strip html
    String testUrl = "http://www.nytimes.com/";    

    // fetch the contents of the webpage
    RiHtmlParser fetcher = new RiHtmlParser(this);
    String pgContents = fetcher.fetch(testUrl, stripTags);

    // call your method (written elsewhere) to pull-out the sentences
    List sents = extractSentences(pgContents);

    // then print them out...
    for (int i = 0; i < sents.size(); i++)
      println(i+") "+sents.get(i));

    println("Got "+sents.size()+" total sentences.");

Note: One (of many) good references on regular expressions is the Pattern class in the Java API itself.


2) Write a regular expression that matches all binary strings (containing only 0s & 1s) of length at least 1, where no digit repeats, (e.g, 0, 1, 10, 010, 10101, 01010101, etc.)



3) Same as above, but the string must be at least 3 digits long and have exactly 1 repetition of a digit (e.g, 0110, 100101, 100, 010101011, 110, etc.). This one is a bit tricky, but do give it a try (after solving #2). Extra credit for the most concise correct expression.


No need to code #2-3, just submit the regular expressions in your email (no back-references, look-aheads, etc. needed).
