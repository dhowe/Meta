N-Gram projects

Work in natural language is is often categorized into one of two basic camps. The first, exemplified by our work with grammars, relies on humans to create the rules of a system (e.g., your grammar files.) The second approach relies on the statistical analysis of existing texts. From these statistical properties, we can create models (e.g., n-gram models like Travesty [letter-level] or RiMarkov [word-level].) And from these models (which enforce the statistical 'rules' found in our inputs) we can then create interesting new outputs.

In your second mini-projects, as discussed in class, you will use the the latter approach to create a compelling artwork.

As usual, you're required to hand in (at least) 2 working iterations, the first of which should be ready to presented in our next class. Although we focused on the N-Gram (or Markov-chaining) technique in class (see our example sketch and src,) you are free to use other statistical techniques, e.g., Bayesian modeling, Keyword-In-Context-Models, etc.

    Constraints

  • Use language in some way (written, spoken, visualized, synthesized, etc.)
  • Use N-Grams (or some other generative statistical method)
  • Final iterations should be read/performed in class
  • You are encouraged to use additional media, e.g., animation, sound, text-to-speech, images, video, etc.
  • Collaboration is fine; but let me know in advance if you have a collaborative project in mind

Simply colliding several texts and displaying the output is NOT an adequate response to this assignment (though it may well be a good first step)


Note: the RiMarkov object contains a range of methods that extend basic n-gram functionality, so be sure to read through closely through the functions and documentation. If any of this documentation is unclear, please send me a note requesting further explanation''


Note: Do not let long loading times deter you from working with vast amounts of text, if this is your interest (e.g., hundreds of even thousands of novels, or say, an encyclopedia or two.) If you need help working with large numbers of files send an email to me or the class list.


Some Random Ideas

  • Use n-grams to create a text that is continually changing in real-time, either on its own, or in response to user interaction. The getCompletions(before, after); method (see below) could be useful for this.
  • Create an n-gram model from just the syllables from a set of text. Then build a dynamic text only from neologisms (things not in RiLexicon. Or use the min-edit-distance algorithm (also in RiLexicon) to compute the nearest real words and substitute them in.
  • Use a statistical model to generate a specific poetic form, based an meter (stressed/unstressed syllables), or rhyme, or some other feature (See RiAnalyzer.analyze(txt);)
  • Created two n-gram models, one using words, and the other using parts-of-speech. Then figure out how to combine to create stronger outputs with lower n-values (see RiTa.getPosTags(txt);)
  • Use an n-gram model to generate both an output text and an accompanying image, soundtrack, or animation.
  • Ignore all these and come up with your own cool idea!
