Interfaces

An interface is a group of related methods with empty bodies. A bicycle's behavior, if specified as an interface, might appear as follows: 

 interface Bicycle {

         public void changeGear(int newGear);

         public void speedUp(float speedIncrement);

         public void applyBrakes(float speedDecrement);
  }

Notice that an interface does not contain any variables, constructors, or method bodies, just the method signatures. To implement this interface with one of your classes (e.g., AcmeBicycle), you'd use the 'implements' keyword in your class declaration as follows: 

  class AcmeBicycle implements Bicycle {

         //  the class MUST contain the methods 
         //  above, in addition to any others...

  }

Implementing an interface allows a class to be more formal about the behavior it promises to provide. Interfaces form a contract between the class and the outside world, and this contract is enforced by the compiler. If your class claims to implement an interface, ALL the methods listed in that interface must appear in its source code (with method bodies) or the class will not compile.

Abstract Classes

An abstract class is a just like a regular class except that it cannot be instantiated. Generally, abstract classes contain one or more abstract methods, which look just like the ones in the interface example above. The difference is that abstract classes can also contain variables, constructors, and other fully-implemented methods. An example of an abstract class representing a bicycle follows:

abstract class Bicycle { 

         // some variables         
         int gear;
         float speed;
         Color color;

         // a constructor
         Bicycle(Color theColor) {
             color = theColor;
         }

         // a normal method
         public void changeGear(int newGear) {
            gear = newGear;
         }

         // an abstract method
         public abstract void changeSpeed(float percent);

  }

Because the class is abstract, you cannot create (or instantiate) it using 'new'. Instead you must subclass it (via extends), and the subclass must provide 'bodies' for any abstract methods. To subclass this abstract class with one of your own classes (e.g., AcmeBicycle), you'd use the 'extends' keyword in your class declaration as follows: 

  class AcmeBicycle extends Bicycle {

        // the class MUST contain any abstract methods
        // above, in addition to the others you define...

        public void changeSpeed(float percent) {
           speed = speed * percent;
        }

  }

Abstract classes are useful b/c you can put more of the 'guts' in the superclass, as opposed to interfaces, which can only have 'empty' methods. But, in Java, a class can only extend one other class (no multiple inheritance), so interfaces are often useful when your class has already 'extended' some other class (e.g, all of our Processing sketches which extend PApplet).
