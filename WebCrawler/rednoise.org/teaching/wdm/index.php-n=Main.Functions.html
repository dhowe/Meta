Methods (also called 'functions')

"[Methods] break large computing tasks into smaller ones, and enable people to build on what others have done instead of starting over from scratch." -Kernigan & Ritchie

In Processing, we've been using methods (often referred to as 'functions') all along. When we say "line(0,0,200,200)" we are calling the method "line", a 'built-in' function of the processing environment (actually it belongs to the PApplet class, but for now we can consider it 'built-in'.)

When we say "void setup()", we are writing a definition for the "setup" method, a special processing method that is automatically called when the program first starts (other automatically-called Processing methods we've seen include: "draw" and "mouseClicked")

A method, whether 'built-in', or defined by you, has three parts which are specified in its declaration (or definition):

  1. the method name
  2. the return type
  3. the parameters

A method declaration looks like this: 

  ReturnType methodName(parameter-list) 
  {
    the code of the method 
  }

Here's a simple example: 

  void sayHello() 
  {
    println("I'm saying hello!");
  }

This is a simple method that performs one basic task -- printing the phrase "I'm saying hello!"). Its method name is " sayHello". It doesn't return anything so we label its return type as "void". It has no parameters (also called argurments) and so nothing is placed in between the parentheses (Note: the empty parentheses are required). We can call the method (and thus execute its code) like this: 

  void setup() 
  {
    size (300,300);

    // call the method here
    sayHello();

    // do other stuff
  }

Return Types and Parameters

Ok, now let's look at an example that makes use of a return-type and parameters: 

  int add(int a, int b) 
  {
    int sum = 0;
    sum = a + b;

    return sum;
  }

In the first line above (the method declaration), the first word is "int" -- which declares "int" as the "return" type for the method. Second, we have the word "add", which is our name for the method. Much like declaring a variable name, we can use any word we want, staying away from already defined keywords (built-in processing method names, etc.) Finally, we have some stuff in between the parentheses. These are called "parameters" and we can have as many of them as we like (including zero as in the "sayHello" case above). Here we have two ('a' and 'b', both of which are of type "int".

So, what are all these things doing? Consider the following code: 

  int x = 5;
  int y = 7;
  int total = 0;

  total = add(x, y); 

  println(total);

In the above code, the method "add" is being called, with two parameters (x and y) and the result of that method is then returned and assigned to (or stored in) the variable "total".

  • Return-type: The function add has a return-type of "int." This means that at the end of the method, we will have a statement "return x;" where x is some int value to be sent back to the program. We need to store this value in some variable (total in the case above) or else it is lost when the method ends.
  • Parameters: Here we pass x and y to the "add" method, which hold the values 5 and 7, respectively. These are also called the 'arguments.' Parameters passed to a method arrive in the same order as they were sent. The first argument lands in the first parameter, second in the second, and so on. Variables passed as parameters must have the same type as specified in the method definition, that is, x and y (in this case) must be of type "int".
  • Local Variables -- notice that the above method "add" declares another variable called "sum" which it uses to compute the value of (a + b). Such variables are called 'local' variables, because they live only within the function. They are created each time the method is called (or "invoked"), and destroyed when the method finished and 'returns'. Such variables are not accessible to the rest of the program.

Things to remember

  • A method must always have a return type. If it doesn't return anything, we declare the return-type as "void".
  • The value you pass as an argument to a method can be a literal value (20, 4.3, "hello", etc.) or a variable of the specified type (e.g., int, String, RiText, etc.)
  • If the return-type is something other than "void", your method must return a value compatible with (of the same type as) the return-type.
